Introduction

To access the data, you can either use the manager or directly the Eloquent model.

If you wish to extend any class, remember to update the configuration. You can publish the configuration through the command php artisan vendor:publish

Let's first check the list of all attributes

Attributes

Name Fillable Required Unique Default Comment
id No No No / Indentify the entity
name Yes Yes Yes null
description Yes No No null
data_builder_id Yes No No null
filename Yes Yes No null
filetype Yes Yes No null
body Yes No No null
created_at No No No / Indicate the date when the record was created
updated_at No No No / Indicate the date when the record was last updated
deleted_at No No No / Indicate the date when the record was soft-deleted

Model

Base class Railken\Amethyst\Models\FileGenerator

You can extends the class like the following example in app/Models/FileGenerator

namespace App\Models;

use Railken\Amethyst\Models\FileGenerator as Model;

class FileGenerator extends Model {
    // ...
}

Remember to update the configuration with new class.

Manager

Base class Railken\Amethyst\Managers\FileGeneratorManager

The manager is the main class to access and manipulate your model.

Why use the manager instead of the model? Because the manager handle all the boring stuff like validation and authorization for you. Remember that the manager return always a Result.

You can extends the class like the following example in app/Managers/FileGeneratorManager

namespace App\Managers;

use Railken\Amethyst\Managers\FileGeneratorManager as Manager;

class FileGeneratorManager extends Manager {
    // ...
}

Remember to update the configuration with new class.

Create a new entity

First you have to define a new instance of the Manager

use Railken\Amethyst\Managers\FileGeneratorManager;

$manager = new FileGeneratorManager();

Now you can use the method create to create a new a new entity

Result

Once you've got the result you should always check if an error has occurred, if not, retrieve the resource.

use Railken\Amethyst\Managers\FileGeneratorManager;

$manager = new FileGeneratorManager();

$result = $manager->create([
    "name" => "Prof. Evangeline Nikolaus DVM",
    "description" => "Tempore fuga quia eligendi ea sapiente ea tempore. Natus ab enim velit numquam. Aut est consequatur nostrum provident dolores rerum non et.",
    "data_builder" => [
        "name" => "Mrs. Reva Anderson",
        "description" => "Quia dolores omnis facilis est ut. Est minima deserunt saepe illo debitis. Molestiae quo explicabo ut quae.",
        "class_name" => "Railken\\Amethyst\\DataBuilders\\CommonDataBuilder",
        "class_arguments" => "- Railken\\Amethyst\\Managers\\DataBuilderManager\n",
        "filter" => "id eq 1",
        "input" => "date:\n    type: date\n    validation: 'date_format:Y-m-d'\n",
        "mock_data" => "date: '2018-01-01'\n",
        "include" => "bar",
        "data_builder" => [
            "class_name" => "Railken\\Amethyst\\DataBuilders\\FileGeneratorDataBuilder"
        ]
    ],
    "filename" => "users-{{ \"now\"|date(\"Ymd\") }}",
    "filetype" => "text/plain",
    "body" => "test"
]);

if ($result->ok()) {

    $resource = $result->getResource();

} else {

    // Loop through all errors
    $result->getErrors()->map(function($error) {
        return $error->toArray();
    }))

    // Retrieve an array of all errors
    $result->getSimpleErrors();

    /* The result is something like this:

        [0] => Array
            (
                [code] => FIELD_NOT_DEFINED
                [label] => field
                [message] => The field is required
                [value] =>
            )
    */

}

The schema is used to define the structure of the attributes. All the $attributes in the model and in the manager are initialized by the schema.

You can extends the class like the following example in app/Schemas/FileGeneratorSchema

    // ...
}

Remember to update the configuration with new class.

Repository

Base class Railken\Amethyst\Repositories\FileGeneratorRepository

The repository is the class used to perform queries.

use Railken\Amethyst\Managers\FileGeneratorManager;

$manager = new FileGeneratorManager();

$repository = $manager->getRepository();

Retrieving an entity

$repository->findOneBy(['id' => 1]);
$repository->findOneById(1);

Retrieving all entities

$repository->findAll();

Performing a query using \Illuminate\DataBase\Eloquent\Builder

$repository->newQuery()->where('id', 1)->first();

You can extends the class like the following example in app/Repositories/FileGeneratorRepository

namespace App\Repositories;

use Railken\Amethyst\Repositories\FileGeneratorRepository as Repository;

class FileGeneratorRepository extends Repository {
    // ...
}

Remember to update the configuration with new class.
